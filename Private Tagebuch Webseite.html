Hier ist eine einfache Möglichkeit, wie du eine private Webseite als Tagebuch erstellen kannst, ohne dass sie öffentlich indexiert wird. Du brauchst dafür nur grundlegende HTML-Kenntnisse und kannst die Datei lokal auf deinem Computer speichern.

📝 Schritt-für-Schritt-Anleitung

1. HTML-Datei erstellen

Erstelle z. B. eine Datei namens tagebuch.html mit folgendem Inhalt:

html
<!DOCTYPE html>
<html lang="de">
<head>
    <meta charset="UTF-8">
    <title>Mein Tagebuch</title>
    <meta name="robots" content="noindex, nofollow"> <!-- Verhindert Indexierung -->
    <style>
        body {
            font-family: Arial, sans-serif;
            max-width: 800px;
            margin: auto;
            padding: 2em;
            background-color: #f4f4f4;
        }
        textarea {
            width: 100%;
            height: 200px;
        }
        .eintrag {
            background: #fff;
            padding: 1em;
            margin-top: 1em;
            border-radius: 8px;
        }
    </style>
</head>
<body>
    <h1>Mein persönliches Tagebuch</h1>
    <form id="tagebuchForm">
        <label for="eintrag">Neuer Eintrag:</label><br>
        <textarea id="eintrag" required></textarea><br>
        <button type="submit">Speichern</button>
    </form>

    <h2>Einträge</h2>
    <div id="eintraege"></div>

    <script>
        const form = document.getElementById('tagebuchForm');
        const eintraegeDiv = document.getElementById('eintraege');
        let eintraege = JSON.parse(localStorage.getItem('tagebuchEintraege')) || [];

        function renderEintraege() {
            eintraegeDiv.innerHTML = '';
            eintraege.slice().reverse().forEach((eintrag, index) => {
                const div = document.createElement('div');
                div.className = 'eintrag';
                div.textContent = `[${eintrag.datum}] ${eintrag.text}`;
                eintraegeDiv.appendChild(div);
            });
        }

        form.addEventListener('submit', function(e) {
            e.preventDefault();
            const text = document.getElementById('eintrag').value;
            const datum = new Date().toLocaleString();
            eintraege.push({ text, datum });
            localStorage.setItem('tagebuchEintraege', JSON.stringify(eintraege));
            document.getElementById('eintrag').value = '';
            renderEintraege();
        });

        renderEintraege();
    </script>
</body>
</html>

🔍 Warum passiert das?

Das Skript in der HTML-Datei speichert deine Einträge im sogenannten localStorage des Browsers. Das bedeutet:

  • Die Daten sind an die Domain und den Pfad gebunden, also z. B. file:///C:/.../tagebuch.html.

  • Wenn du die Datei anders benennst, verschiebst oder mit einem anderen Browser öffnest, sieht der localStorage leer aus.

  • Manche Browser (vor allem Firefox und Safari) blockieren oder löschen localStorage, wenn die Datei lokal (file://) geöffnet wird.

  • Browser im „Privaten Modus“ speichern ebenfalls nichts dauerhaft.

✅ Lösungen

1. Verwende immer denselben Pfad und Browser

  • Öffne die Datei immer aus demselben Ordner und immer mit demselben Browser.

  • Vermeide es, die Datei zu verschieben oder umzubenennen, wenn du Daten behalten willst.

2. Verwende einen lokalen Webserver (empfohlen)

Wenn du willst, dass alles stabil läuft (auch langfristig), richte dir einen kleinen lokalen Webserver ein:

So geht’s (einfachste Variante mit Python):

  1. Öffne den Ordner, in dem deine tagebuch.html liegt.

  2. Starte einen Server mit:

    bash
    python -m http.server 8000

  3. Öffne dann im Browser:

    bash
    http://localhost:8000/tagebuch.html

Jetzt wird localStorage richtig funktionieren, als wäre es eine „echte Webseite“.

3. Alternativen zur Speicherung:

  • Statt localStorage kannst du auch:

    • Die Einträge in eine Textdatei speichern (erfordert z. B. Electron oder ein Python-Backend).

    • Ein Desktop-Tool verwenden, z. B. Obsidian, Logseq oder Joplin, das genau für sowas gebaut wurde.

💡 Fazit

Wenn du es ganz einfach halten willst:

  • Nutze immer denselben Browser

  • Öffne die Datei nicht im Inkognito-Modus

  • Ändere den Pfad/Dateinamen nicht

Oder: Starte einen lokalen Webserver – dann läuft’s wie eine kleine Offline-App.

🔧 1. server.py (Python-Backend)

python
from flask import Flask, request, send_from_directory, jsonify
import os
from datetime import datetime

app = Flask(__name__)

EINTRAEGE_DATEI = 'eintraege.txt'

@app.route('/')
def index():
    return send_from_directory('.', 'index.html')

@app.route('/eintragen', methods=['POST'])
def eintragen():
    text = request.form.get('text', '').strip()
    if text:
        mit_datum = f"[{datetime.now().strftime('%Y-%m-%d %H:%M:%S')}] {text}\n"
        with open(EINTRAEGE_DATEI, 'a', encoding='utf-8') as f:
            f.write(mit_datum)
    return '', 204

@app.route('/eintraege')
def eintraege():
    if os.path.exists(EINTRAEGE_DATEI):
        with open(EINTRAEGE_DATEI, 'r', encoding='utf-8') as f:
            zeilen = f.readlines()
    else:
        zeilen = []
    return jsonify(zeilen[::-1])  # neueste zuerst

if __name__ == '__main__':
    app.run(debug=True)

    <script>
        async function ladeEintraege() {
            const res = await fetch('/eintraege');
            const daten = await res.json();
            const container = document.getElementById('eintraege');
            container.innerHTML = '';
            daten.forEach(text => {
                const div = document.createElement('div');
                div.className = 'eintrag';
                div.textContent = text;
                container.appendChild(div);
            });
        }

        document.getElementById('form').addEventListener('submit', async function(e) {
            e.preventDefault();
            const formData = new FormData(this);
            await fetch('/eintragen', { method: 'POST', body: formData });
            this.reset();
            ladeEintraege();
        });

        ladeEintraege();
    </script>
</body>
</html>
